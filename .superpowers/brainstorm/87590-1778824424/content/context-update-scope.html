다음 추천에 바로 반영되는 컨텍스트

저장 완료 후 받은 답변은 단순 피드백이 아니라, 다음 AI 추천 전에 조회되는 context로 업데이트됩니다.

현재 구조에 추가될 흐름
1. 거래 저장
Event + Transaction 저장 완료
↓
2. 변경 이유 질문
선택 칩 + 직접 입력으로 사용자 선택 맥락 수집
↓
3. 관계 context 업데이트
방향, 차액, 이유 코드, 직접 입력을 저장해 다음 추천에서 즉시 조회
확정된 저장 단위
개별 상대방 단위 context
예: 김민수에게는 사용자가 현금 준비 금액을 자주 고려해 추천보다 낮춰 저장했다.
다음에 김민수 이름으로 AI 분석이 들어오면 행사 유형과 무관하게 이 context를 먼저 조회합니다.
