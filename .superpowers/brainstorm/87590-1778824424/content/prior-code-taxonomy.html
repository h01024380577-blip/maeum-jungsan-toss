AI prior code 분류 방식

자유문장은 저장하고, AI가 그 원문을 구조화 prior code로 분류합니다. 다음 추천에는 원문이 아니라 code만 들어갑니다.

고정 code 체계
권장
AI는 정해진 reason code 중에서만 선택합니다.
budget_constraint cash_available companion_count pre_agreed_amount attendance_mode

장점: 데이터가 깨끗하고 다음 추천 로직에 바로 연결하기 쉽습니다.

AI가 새 code 생성
위험
AI가 매번 새로운 code 이름을 만들 수 있습니다.
spouse_attended with_partner couple_amount

문제: 같은 의미가 여러 code로 쪼개져 추천 prior로 쓰기 어려워집니다.

하이브리드 안전장치
고정 code로 분류하고, 맞는 code가 없으면 unknown_context로 저장
원문은 별도 보관하므로 나중에 taxonomy를 확장할 수 있지만, 운영 중인 추천 로직에는 검증된 code만 들어갑니다.
