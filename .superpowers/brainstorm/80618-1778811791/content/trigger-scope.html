언제 “금액 변경 이유”를 물을까요?

현재 저장 흐름: AI가 추천한 금액이 분석 결과 바텀시트에 표시되고, 사용자가 금액을 수정한 뒤 저장할 수 있습니다.

예시 상황
AI 분석 완료
AI 추천 금액
10만원
지난번과 비슷한 금액으로 추천했어요
사용자 저장 금액
7만원
저장 완료 후 이유 질문 대상
A

최종 금액이 AI 추천과 다를 때만

초기 AI 금액과 저장 금액이 다르면 질문합니다. +1만 후 다시 -1만으로 원복하면 묻지 않습니다.

B

사용자가 금액 컨트롤을 건드리면

최종 금액이 같아도 사용자가 금액을 조정했다면 질문합니다. “고민 흔적”까지 수집합니다.

C

AI 분석 저장마다 항상 짧게

금액 변경 여부와 무관하게 저장 후 이유/확신도를 묻습니다. 데이터는 많지만 흐름이 무거워집니다.
