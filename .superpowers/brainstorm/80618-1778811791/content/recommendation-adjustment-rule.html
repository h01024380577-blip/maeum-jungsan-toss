다음 추천 반영 방식

상대방 context가 생긴 뒤, 다음 AI 추천 금액을 어떻게 바꿀지 정해야 합니다.

A. 기존 추천 + context 보정
기본 추천 10만
김민수 context: 지난번 추천보다 2만 낮춤
다음 추천 8만

기존 추천 로직은 유지하고, 상대방 context를 보정값처럼 즉시 반영합니다.

B. 마지막 선택 금액 우선
기본 추천 10만
김민수 마지막 저장 금액: 7만
다음 추천 7만

가장 즉각적이지만, 한 번의 특수 상황이 계속 반복될 위험이 있습니다.

C. 이유별 규칙으로 직접 조정
현금 준비 금액 → -1만 동행 인원 → +1만 약속한 금액 → 해당 금액 우선

이유마다 추천 조정 규칙을 명시합니다. 설명은 좋지만, 규칙 설계가 더 많이 필요합니다.

A

기존 추천 + context 보정

기존 AI prior를 유지하면서 사용자 선택 prior를 즉시 보정값으로 더합니다.

B

마지막 선택 금액 우선

같은 상대방이면 직전 사용자가 저장한 최종 금액을 다음 추천의 중심값으로 둡니다.

C

이유별 규칙 직접 적용

각 이유 코드가 추천 금액을 어떻게 바꾸는지 명시적인 룰을 둡니다.
